master-key recovery, but left Layer 9 of the 9-layer defense contract listed
as **"partial — per-access audit hook deferred"**. 0.9.1 closes that gap.

implementation on `main`:

## What's in 0.9.1

### Layer 9 — `AuditEvent` + `AuditSink`

New module: [`key_vault::audit`].

```rust
use key_vault::{AuditEvent, AuditSink, KeyVaultBuilder, RawKey};
use std::sync::{Arc, Mutex};

// A trivial sink that records events into an in-memory buffer.
#[derive(Default)]
struct InMemoryAudit(Mutex<Vec<AuditEvent>>);

impl AuditSink for InMemoryAudit {
    fn on_event(&self, event: &AuditEvent) {
        // Sink contract: non-blocking, no panics, no back-pressure.
        if let Ok(mut buf) = self.0.lock() {
            buf.push(event.clone());
        }
    }
}

# fn main() -> Result<(), key_vault::Error> {
let sink: Arc<dyn AuditSink> = Arc::new(InMemoryAudit::default());
let vault = KeyVaultBuilder::new()
    .normalize_with_blake3(false)
    .with_audit_sink(Arc::clone(&sink))
    .build();

let handle = vault.register("session", RawKey::new(b"session key".to_vec()))?;
vault.with_key(handle, |_bytes| { /* use the key */ })?;
vault.unregister(handle)?;
// `sink` now holds three events: Register, Read, Unregister.
# Ok(()) }
```

What ships:

- **`AuditEvent`** — `{ timestamp: Duration, key_name: String, kind: AccessKind, thread_id: ThreadId, note: Cow<'static, str> }`. `#[non_exhaustive]` — additive fields (caller identity, request id correlation) are allowed in minor releases.
- **`AccessKind`** — covers `Register`, `Unregister`, `Read`, `Rotate`, `OneShotFragment`, `OneShotDefragment`, and `MasterUnlockAttempt { matched: bool }`. `#[non_exhaustive]`.
- **`AuditSink` trait** — `Send + Sync`; contract is **non-blocking, no panics, no back-pressure into the vault**. Implementors that need to do network or disk work must do it on a background worker.
- **Blanket impl `AuditSink for Arc<dyn AuditSink>`** so callers can pre-wrap a sink and pass it both into the builder and out to their own observers.
- **`NoAudit`** — the default sink. Events are still constructed and dispatched; the cost is one struct construction + one virtual call per op.
- **`LogAudit`** — `tracing`-backed sink (gated on the `monitor-tracing` Cargo feature). Emits `info!` events on the `key_vault::audit` target with structured fields (`key_name`, `kind`, `thread_id`, `timestamp_ms_since_epoch`, `note`).

### Wiring into the vault

`VaultInner` gained an `audit: Arc<dyn AuditSink>` field; every public
operation that touches a key emits an event before returning:

| Operation | `AccessKind` emitted |
|-----------|----------------------|
| `KeyVault::register` | `Register` |
| `KeyVault::unregister` | `Unregister` |
| `KeyVault::with_key` | `Read` |
| `KeyVault::rotate` | `Rotate` |
| `KeyVault::fragment` (one-shot) | `OneShotFragment` |
| `KeyVault::defragment` (one-shot) | `OneShotDefragment` |
| `KeyVault::unlock_with_master` | `MasterUnlockAttempt { matched }` |

The vault constructs the event on the stack, calls `sink.on_event(&event)`,
then drops the event. There is no allocation on the happy path beyond the
`String` for the key name (which the vault already owns).

### Builder API

```rust
use key_vault::{KeyVaultBuilder, LogAudit};

let _vault = KeyVaultBuilder::new()
    .with_audit_sink(LogAudit::new())
    .build();
```

`KeyVaultBuilder::with_audit_sink<A: AuditSink + 'static>(self, sink: A) -> Self`
accepts any `AuditSink`. The blanket `Arc<dyn AuditSink>` impl also lets you
hand the builder a pre-wrapped trait object directly.

### Redaction discipline

`AuditEvent` carries **only the key name** — never key material. The vault
never passes raw key bytes to a sink under any circumstance. The
caller-supplied `note` field is `Cow<'static, str>`; callers are responsible
for keeping it free of secrets if they pass user-supplied strings.

For master-unlock attempts the reserved name `"<master>"` is used; the
`matched: bool` field reports the constant-time comparison result.

## Stability

- **Public API stability** — `audit::*` items follow the same pre-1.0 policy
  as the rest of the crate (subject to change without notice). All public
  enums and structs in the new module are `#[non_exhaustive]`.
- **Backwards compatibility** — adding `with_audit_sink` is purely additive;
  every existing 0.9.0 caller compiles unchanged. The default sink remains
  `NoAudit`, matching prior behavior (events constructed but not routed).
